# SearXNG Local Service

Local SearXNG runs as a rootless Podman Quadlet user service.

## Files

- `quadlet/searxng.container`: container unit generated as `searxng.service`.
- `quadlet/searxng-cache.volume`: persistent Podman volume for `/var/cache/searxng`.
- `etc/settings.yml`: SearXNG settings mounted read-only into `/etc/searxng`.
- `~/.local/state/searxng/searxng.env`: host-local secret and base URL generated by Home Manager activation.

## Commands

```bash
systemctl --user daemon-reload
systemctl --user start searxng.service
systemctl --user status searxng.service
journalctl --user -u searxng.service -f
```

Verify the JSON API used by agents:

```bash
curl 'http://127.0.0.1:8888/search?q=searxng&format=json'
```

If the service should continue without an active login session:

```bash
loginctl enable-linger "$USER"
```
